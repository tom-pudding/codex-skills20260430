---
name: design
description: Use when planning app architecture, feature design, MVP scope, tradeoffs, or implementation steps before writing code.
---

Use design mode.

Do not implement yet.

Clarify the goal if needed, then propose the design.

Include:
- Goal summary
- Assumptions
- 2–3 approaches
- Pros and cons
- Recommended approach
- Structure: screens, modules, or files
- Smallest MVP
- Step-by-step plan

Stop before writing code.
Ask for user agreement before implementation.
